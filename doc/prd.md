# IPMS 프론트엔드 개발 기획서 (PRD)

## 1. 개요 (이 프로젝트는 무엇인가요?)
이 문서는 제휴 관리 시스템(IPMS)의 화면을 어떻게 만들지 정리한 문서입니다.
우리는 **프로젝트 목록을 보여주는 화면**을 가장 먼저 만들 것입니다.
마치 엑셀 표처럼 프로젝트들의 정보를 한눈에 볼 수 있는 화면이라고 생각하시면 됩니다.

## 2. 개발 도구 (어떤 도구로 만드나요?)
우리는 최신 웹 기술을 사용하여 빠르고 부드러운 화면을 만들 예정입니다.
- **Vue.js (뷰 제이에스)**: 화면을 레고 블록처럼 조립해서 만드는 도구입니다.
- **TypeScript (타입스크립트)**: 코드에 오류가 없는지 미리 체크해주는 안전장치입니다.
- **Vite (비트)**: 개발한 화면을 빠르게 미리 보여주는 도구입니다.
- **Axios (액시오스)**: 서버(데이터 저장소)에서 데이터를 가져오거나 보낼 때 사용하는 배달부 같은 역할을 합니다.

## 3. 화면의 생김새 (어떻게 생겼나요?)
화면은 크게 4가지 조각으로 나뉩니다.

| 이름 | 위치 | 설명 |
| --- | --- | --- |
| **헤더 (Header)** | 맨 위 | 로고가 있고, 로그인 정보 등이 표시되는 줄입니다. |
| **메뉴 (Menu)** | 왼쪽 | 원하는 페이지로 이동할 수 있는 버튼들이 모여 있습니다. |
| **본문 (Main Window)** | 가운데 | 실제 작업하는 공간입니다. 예를 들어 '프로젝트 리스트'를 누르면 여기에 표가 나옵니다. |
| **푸터 (Footer)** | 맨 아래 | 저작권 표시나, 회사 주소 같은 부가 정보가 있는 줄입니다. (보통 작게 표시됨) |

## 4. 우리가 만들 첫 번째 화면: "프로젝트 리스트"
이 화면은 등록된 프로젝트들을 검색하고 목록으로 보는 화면입니다.

### 4.1 검색 창 (원하는 프로젝트 찾기)
화면 위쪽에는 검색할 수 있는 상자가 있습니다.
- **프로젝트명 입력칸**: 찾고 싶은 프로젝트 이름을 직접 씁니다.
- **구분 선택 상자**: '전체', '운영', '개발' 중 하나를 선택해서 봅니다.
- **조회 버튼**: 이 버튼을 누르면 검색 조건에 맞는 프로젝트를 찾아옵니다.

### 4.2 목록 표 (데이터 보여주기)
검색된 프로젝트들이 표 형태로 나옵니다. 보여줄 정보는 다음과 같습니다.
(서버에서 가져온 데이터를 그대로 보여줍니다)

- **순번**: 1, 2, 3... 번호
- **구분**: 이 프로젝트가 운영 중인지, 개발 중인지
- **프로젝트명**: 프로젝트의 이름
- **주관사**: 어디서 주관하는지 (예: 삼성, LG 등)
- **기간**: 언제 시작해서 언제 끝나는지 (시작일, 종료일)
- **상태**: 현재 잘 진행되고 있는지 상태
- **위치**: 어디서 일하는지
- **내용**: 프로젝트에 대한 자세한 설명

### 4.3 데이터 명세 (서버와 주고받는 약속)
개발자들은 서버와 데이터를 주고받을 때 서로 약속을 정합니다. 이를 "데이터 모델"이라고 합니다.
우리는 `PrjDto`라는 이름의 보따리에 다음 정보를 담아서 주고받기로 했습니다.

- `prj_id`: 프로젝트의 고유 주민등록번호 같은 것
- `prj_nm`: 프로젝트 이름
- `prj_gbn`: 프로젝트 구분 (운영/개발)
- `prj_str_dt`, `prj_end_dt`: 시작 날짜와 종료 날짜
- `office_loc`: 사무실 위치
- `biz_prtnr`: 파트너 회사 이름

## 5. 앞으로의 계획
이 문서를 바탕으로 실제 화면을 코딩하기 시작할 것입니다.
1. 빈 도화지(프로젝트 생성)를 준비합니다.
2. 헤더, 메뉴 같은 기본 틀을 먼저 만듭니다.
3. 그 틀 안에 '프로젝트 리스트' 표를 집어넣습니다.
4. 마지막으로 '조회' 버튼을 누르면 가짜 데이터라도 뜨게 만들어서 테스트합니다.
