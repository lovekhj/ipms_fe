# IPMS 프론트엔드 개발 계획서 (Implementation Plan)

## 1. 목표 (Goal)
IPMS(제휴 관리 시스템)의 첫 화면을 만드는 것이 목표입니다.
Vue.js라는 도구를 사용해서, 사용자가 모니터에서 볼 수 있는 '프로젝트 목록' 화면을 만들 것입니다.

## 2. 사전 확인 필요 (Checklist)
> [!IMPORTANT]
> **중요하게 확인해야 할 것들**
> - **Vue 버전**: 최신 버전인 Vue 3를 사용한다고 가정했습니다.
> - **디자인 도구**: 화려한 디자인 도구 대신, 가장 기초적인 CSS(스타일 시트)를 사용해 직접 꾸밀 것입니다.
> - **데이터 주소**: 화면에 뿌려줄 데이터를 어디서 가져올지(서버 주소) 확인이 필요하지만, 일단은 가짜 데이터로 만들겠습니다.

## 3. 변경 예정 사항 (Proposed Changes)

### 문서 작업 (Documentation)
이미 `doc` 폴더에 기획서(PRD)를 만들었습니다. 이 계획서도 `doc` 폴더에 같이 보관합니다.

### 프로젝트 설정 (Configuration)
#### [수정] `package.json`
- `axios`라는 도구를 설치 목록에 추가합니다. (서버와 대화하기 위한 전화기 같은 도구입니다)

### 소스 코드 작성 (Source Code)

#### [신규] 기본 레이아웃 (`src/layouts/DefaultLayout.vue`)
- 화면의 전체 틀을 잡는 파일입니다. 헤더, 메뉴, 본문을 어떻게 배치할지 정합니다.

#### [신규] 공통 부품들 (`src/components/common/`)
- **`TheHeader.vue`**: 화면 맨 위의 헤더 부분
- **`TheFooter.vue`**: 화면 맨 아래의 푸터 부분
- **`TheMenu.vue`**: 왼쪽의 메뉴 부분
레고 블록처럼 이 부품들을 만들어서 조립할 것입니다.

#### [신규] 프로젝트 리스트 화면 (`src/views/project/ProjectList.vue`)
- 우리가 목표로 하는 진짜 알맹이 화면입니다.
- 검색창과 표(Grid)가 들어갑니다.

#### [신규] 데이터 통신 (`src/api/project.ts`)
- 서버에서 데이터를 가져오는 심부름꾼 파일입니다.
- `getPrjList`라는 함수를 통해 프로젝트 목록을 가져오게 시킬 것입니다.

## 4. 확인 방법 (Verification Plan)
### 자동 테스트
- 컴퓨터가 자동으로 코드를 실행해보고 에러가 없는지 체크합니다. (`npm run dev` 명령어를 사용)

### 직접 눈으로 확인하기
1. 화면이 기획서의 그림(와이어프레임)과 비슷한지 봅니다.
2. 메뉴를 눌렀을 때 화면이 잘 바뀌는지 봅니다.
3. 조회 버튼을 눌렀을 때 표에 데이터가 잘 나오는지 확인합니다.
